# Product Vision: TasK

## 1. Vision Statement (Шаблон 2.1)

**Для** команд разработки и бизнеса, **у которых проблема в** сложности и трудоемкости извлечения структурированных знаний из больших объемов неструктурированных данных (сложные PDF, аудио-записи, веб-источники), **наш продукт TasK** **поможет тем, что** станет надежным и масштабируемым конвейером (pipeline) автоматической обработки, анализа и индексации информации, **и подойдет лучше предложений конкурентов** (таких как разрозненные скрипты или закрытые SaaS-решения) **потому что** объединяет мощь современных AI-инструментов с промышленной надёжностью enterprise-класса, обеспечивая полный контроль над данными и процессом их обработки.

---

## 2. Elevator Pitch (Презентация в лифте)

"Представьте, что тысячи ваших документов, отчетов и записей совещаний превратились в живую базу знаний, которая сама отвечает на вопросы. **TasK** — это интеллектуальный конвейер, который берет на себя всю 'грязную' работу: он сам скачивает данные, распознает сложные PDF, транскрибирует аудио и видео с **идентификацией спикеров**, и готовит информацию для AI. В отличие от закрытых SaaS-решений, TasK разворачивается на вашей инфраструктуре — полный контроль над данными, никаких ежемесячных подписок и независимость от внешних провайдеров."

---

## 3. Design the Box (Спроектируй коробку)

### Лицевая сторона (Front)
*   **Заголовок:** TasK: Knowledge Engineering Platform
*   **Слоган:** От хаоса документов к интеллекту данных.
*   **Визуальный образ:** Стилизованный конвейер, в который входят стопки бумаг, аудиозаписи и видеоролики, а на выходе получается светящийся кристалл (структурированные данные), подключенный к нейросети.

### Оборотная сторона (Back)
**TasK** — это ваш персональный завод по переработке данных в знания.

**Ключевые возможности:**
*   **Smart Ingestion:** Автоматический сбор данных из облачных хранилищ, веб-ресурсов и корпоративных систем.
*   **Deep Processing:** Высокоточное извлечение смыслов из сложных документов и медиа-контента.
*   **Speaker Diarization:** Идентификация спикеров в аудио/видео — извлечение экспертных мнений с привязкой к автору.
*   **Semantic Search:** Мгновенный поиск «по смыслу», а не по ключевым словам, по всей базе знаний компании.
*   **Reliable Engineering:** Промышленная архитектура, гарантирующая сохранность данных и масштабируемость под любые задачи.

**Результат:** Ваша компания получает фундамент для внедрения умных ассистентов на собственных данных без риска утечки информации.

*   **Через полгода (2026-08-25)** мы предоставим пользователям возможность подключать любой S3-архив и получать полностью проиндексированную базу знаний за один клик.

## 5. Problem Statement

Проблема, которую решает TasK:

1. **Информационная перегрузка**: Компании накапливают огромные объёмы неструктурированных данных (PDF, аудио-записи, видео, заметки), но не могут эффективно их использовать.
2. **Трудности поиска**: Поиск по содержимому документов часто ограничивается ключевыми словами и не учитывает смысл.
3. **Ручной труд**: Извлечение знаний из документов требует значительных человеческих усилий.
4. **Разрозненные решения**: Использование отдельных Python-скриптов или закрытых SaaS-сервисов не даёт целостной картины контроля над данными.

## 6. Target Users

### Primary Users
- **Internal Team**: разработчики и инженеры проекта TasK — первые пользователи для тестирования и обратной связи
- **Developers**: технические специалисты, которые интегрируют TasK в свои системы через API
- **Knowledge Workers**: сотрудники, работающие с большими объёмами документации и требующие быстрого поиска

### Secondary Users
- **Product Managers**: создают и настраивают проекты для команд
- **Data Analysts**: используют RAG-возможности для аналитики

---

## 7. Value Proposition

Для пользователей TasK предоставляет:

1. **Время**: автоматизация рутинной обработки данных освобождает время для высокоуровневых задач
2. **Качество**: AI-first подход обеспечивает более точное извлечение смыслов, чем ключевое слово
3. **Суверенитет**: self-hosted и локальная обработка обеспечивают приватность и контроль над данными, а также независимость от vendor lock-in
4. **Интеграция**: open API и открытые стандарты позволяют встраивать TasK в существующие рабочие процессы

---

## 8. Product Principles

1. **AI-First**: AI не надстройка, а фундаментальная часть системы
2. **Privacy-First**: данные пользователя остаются под его контролем; self-hosted/on-premise развёртывание и независимость от vendor lock-in
3. **Developer-Friendly**: clean API, документация, возможность self-host и интеграции через открытые стандарты
4. **Markdown-First**: предпочтение markdown-формату для хранения и работы с контентом
5. **Extensibility**: плагины для разных источников данных и форматов

---

## 9. Non-Goals

Что TasK точно **НЕ** делает:

- Не конкурирует с Notion/Obsidian как редактор документов — мы фокусируемся на processing и RAG
- Не предоставляет социальные фичи — комментарии, рецензии, совместное редактирование
- Не разрабатывает мобильные приложения — фокус на web/API/CLI
- Не хостит данные пользователей как SaaS — priority: self-hosted deployment
- Не является универсальной CRM/ERP системой — мы специализируемся на knowledge engineering

## 10. Success Metrics

### Ключевые метрики
- **Documents processed**: количество успешно обработанных документов
- **Search success rate**: доля успешных поисковых запросов (relevant results)
- **API usage**: количество запросов к API (среднее, p95, p99)
- **Indexing time**: среднее время индексации документа

### Metabolic metrics
- **Retention rate**: доля пользователей, продолжающих использовать TasK
- **Query latency**: latency ответа на поисковый запрос

## 11. Scope Boundaries

### Что ВХОДИТ в TasK

- Ingest источников (S3, HTTP, локальные файлы)
- Текстовое извлечение (PDF, DOCX, TXT)
- Транскрибация аудио/видео
- **Аудио-диаризация (speaker identification)**
- Chunking и embeddings для RAG
- Vector search (semantic search)
- API для работы с индексом
- CLI для ingestion

### Что НЕ входит в TasK

- OCR-распознавание сканированных изображений (handwriting)
- Полноценный NLP-анализ (sentiment, NER, etc.)
- Управление доступом на уровне документа
- Визуализация данных/дэшборды
- Формирование отчётов и аналитика
- Интеграция с Jira/Trello/etc.
- Мобильные приложения
- SaaS-хостинг данных

## 12. Glossary

| Термин | Описание |
|--------|----------|
| **RAG** | Retrieval-Augmented Generation — подход, при котором LLM использует внешние данные для генерации ответов |
| **Chunking** | Разбиение текста на фрагменты (chunks) для последующей векторизации |
| **Vector DB** | База данных для хранения и поиска векторов (embeddings) |
| **Embeddings** | Векторные представления текста, отражающие смысл |
| **Source** | Источник данных для ingestion (S3 bucket, URL, локальный файл) |
| **Index** | Индексированное представление данных для search |
| **MVP** | Minimum Viable Product — минимальная версия продукта для внутреннего использования |
| **MMP** | Minimum Marketable Product — минимальная версия для выхода на рынок |
| **Ingest** | Процесс загрузки и первичной обработки данных из источника |
| **Query** | Поисковый запрос к системе |
| **Agent** | Автономный компонент, выполняющий задачи (например, обработка документа) |
| **Diarization** | Идентификация спикеров в аудио/видео — разделение потока речи по говорящим с привязкой фрагментов к конкретным лицам |
