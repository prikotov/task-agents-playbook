# Деплой (Deploy)

**Деплой (deploy)** — публикация изменений на production-сервере с необходимыми действиями (обновление кода, установка зависимостей, миграции, прогрев кэша, перезапуск/перезагрузка сервисов).

## Границы ответственности

- Документ описывает только общие правила и ссылки на актуальные runbook.
- Не выполняй деплой без явного запроса пользователя/ответственного инженера.
- Не выполняй миграции и любые действия на production без явного запроса.
- В рамках PR фиксируй только изменения кода/доков.
- В рамках PR фиксируй инструкции и чеклисты.
- В рамках PR фиксируй риски и порядок действий.

## Правила

- Деплой выполняется только из `master` (после merge PR).
- Если релиз включает миграции — деплой должен учитывать порядок действий и риски (окна несовместимости, откат).

## Рекомендуемый поток (высокий уровень)

1. Merge PR в `master` (после `Code Review`).
2. Выпустить релиз (если применимо) по гайду релизов.
3. Выполнить деплой по актуальному runbook (см. ссылки выше).
4. Провести пост-проверку: основные user-flow, логи, метрики.
