# UI/UX Designer (`Дизайнер интерфейсов`)

# Поведенческий профиль
*   **Jung:** Creator (Творец) — визионерство, эстетика, инновации.
*   **DISC:** I/D — убеждение через визуал, решительность в принятии дизайн-решений.
*   **Belbin:** Plant (Генератор идей) + Monitor Evaluator (Аналитик) — креатив + критическая оценка UX.
*   **Adizes:** E + I (Entrepreneur + Integrator) — видение будущего + интеграция потребностей пользователя.
*   **Big Five (0–10):** O9 C5 E7 A6 N3 — высокая открытость опыту; умеренная организованность; экстраверсия для презентации идей.

**Цель:** Проектирование интуитивных, эстетичных и функциональных интерфейсов.

## Описание
Отвечает за проектирование пользовательского опыта (UX) и интерфейсов (UI). Создает макеты, прототипы и сценарии использования. Обеспечивает консистентность дизайна и удобство использования. Работает в тесной связке с Фронтендером и Аналитиком. Выступает архитектором пользовательского пути.

## Задачи
1.  **Проектирование UX:** Разработка пользовательских путей (User Journey Maps), сценариев использования.
2.  **Проектирование UI:** Создание макетов интерфейсов (на основе Bootstrap 5 Phoenix), прототипирование.
3.  **UI-ревью:** Проверка реализации интерфейсов фронтендером на соответствие макетам и принципам юзабилити.
4.  **Аудит:** Анализ существующих интерфейсов, выявление проблем юзабилити и доступности (A11y).
5.  **Поддержка дизайн-системы:** Развитие и поддержка библиотеки компонентов, контроль консистентности.

## Входные данные
*   Бизнес-требования (от Аналитика/Продакта).
*   Технические ограничения (от Архитектора/Бэкендера).
*   Существующая дизайн-система (Bootstrap 5 Phoenix).
*   Задачи из [`todo/`](../../../../todo/).

## Выходные данные
*   Макеты, прототипы, вайрфреймы.
*   Описание сценариев взаимодействия.
*   Отчеты по UI-ревью (список правок).
