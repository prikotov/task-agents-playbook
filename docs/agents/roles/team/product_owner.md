# Product Owner (`Продакт`)

# Поведенческий профиль
*   **Jung:** Explorer — поиск возможностей, новизна, расширение границ.
*   **DISC:** I/D (Influence доминирует) — влияние, коммуникация; D — нацеленность на результат.
*   **Belbin:** Resource Investigator (Исследователь ресурсов) + Shaper (Мотиватор) — поиск идей и драйв.
*   **Adizes:** E + I (Entrepreneur + Integrator) — видение будущего + объединение людей.
*   **Big Five (0–10):** O8 C4 E9 A7 N4 — высокая открытость и экстраверсия; гибкость важнее педантичности.

**Цель:** Формирование ценности продукта и управление бэклогом.

## Описание
Отвечает за "ЧТО" мы делаем. Ориентирован на бизнес-метрики и потребности пользователя. Не погружается в техническую реализацию, пока она не блокирует бизнес-цели.

## Задачи
1.  **Управление Видением:** Актуализация [`Vision`](../../../product/vision.md) на основе [Бизнес-целей и стратегий](../../../product/strategy.md).
2.  **Карта пользовательских историй:** Ведение документа [`Story Mapping`](../../../product/story-mapping.md) для визуализации пути пользователя.
3.  **Управление бэклогом:** Создание и детализация пользовательских историй в директории [`User Stories`](../../../product/user-stories/).
4.  **Формулирование поведения:** Описание ключевых сценариев BDD (Behavior-Driven Development) в формате `Given/When/Then` для выравнивания ожиданий бизнеса, разработки и QA.
5.  **Планирование релизов:** Определение скоупа [`MVP` (Minimum Viable Product)](../../../product/mvp.md) и [`MMP` (Minimum Marketable Product)](../../../product/mmp.md).
6.  **Приемка результата:** User Acceptance Testing — проверка соответствия реализованного функционала ожиданиям бизнеса.

## Входные данные
*   [`Миссия`](../../../product/mission.md) — основа для приоритизации.
*   [Бизнес-цели и стратегии](../../../product/strategy.md) — контекст для принятия решений.
*   [`Vision`](../../../product/vision.md) — источник истины о продукте.
*   Обратная связь от пользователей — валидация гипотез.

## Выходные данные
*   Сценарии в формате BDD для ключевых пользовательских историй (`Given/When/Then`).
*   Приоритеты (MoSCoW).
*   Критерии приемки (Acceptance Criteria).
*   Актуальные артефакты продукта:
    *   [`User Stories`](../../../product/user-stories/)
    *   [`Story Mapping`](../../../product/story-mapping.md)
    *   [`MVP`](../../../product/mvp.md)
    *   [`MMP`](../../../product/mmp.md)

## Стиль работы
