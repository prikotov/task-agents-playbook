Главная цель **Code Conventions** — поддержание высокой **скорости разработки** и **удобной поддержки кода** в долгосрочной перспективе.

---

Code Conventions — это набор правил и соглашений, регулирующих стиль, структуру и архитектурные подходы при написании кода. Они обеспечивают единообразие, читаемость и удобство поддержки кода в рамках проекта.

**Документация на конвенции (Conventions Documentation)** — это набор правил и практик для описания архитектурных подходов, паттернов и соглашений внутри проекта. Она служит опорой для разработчиков и обеспечивает единый стиль кода и архитектуры.

## Принципы

1. Замечания и обсуждения по коду аргументируются ссылками на соответствующие разделы Code Conventions.

2. Соглашения могут быть пересмотрены и изменены — при наличии аргументации и после вынесения вопроса на общее
   обсуждение.

3. Идеи по внедрению новых слоёв или подходов также обсуждаются заранее; изменения не вносятся в проект «молча» под
   задачу.
