# Qdrant: Векторная база данных

Qdrant — это высокопроизводительная векторная база данных с открытым исходным кодом, используемая в проекте для задач, связанных с семантическим поиском и индексацией данных для AI-агентов. Она позволяет эффективно находить похожие объекты в больших наборах данных на основе их векторных представлений.

## Назначение

В данном проекте Qdrant используется **ИИ-агентами Kilo Code и RooCode** и выполняет следующие функции:
- **Индексация кодовой базы**: Хранение векторных представлений кода для быстрого поиска релевантных фрагментов при работе AI-агентов.
- **Семантический поиск**: Обеспечение поиска по смысловой близости, а не только по ключевым словам.

## Порты

Контейнер Qdrant открывает два порта:

- **`6333` (HTTP)**: Порт для REST API и доступа к веб-интерфейсу (дашборду). Вы можете открыть `http://localhost:6333` в браузере для мониторинга и управления коллекциями.
- **`6334` (gRPC)**: Порт для gRPC. Используется клиентскими библиотеками для высокопроизводительного взаимодействия с базой данных.

Хотя для работы агентов по индексации кода, скорее всего, будет использоваться только gRPC порт, REST API и дашборд крайне полезны для отладки.

## Управление контейнером

Для управления контейнером Qdrant используются следующие `make` команды:

### `make qdrant-up`

Запускает контейнер Qdrant в фоновом режиме (`detached mode`).

```bash
make qdrant-up
```

### `make qdrant-down`

Останавливает и удаляет контейнер Qdrant.

```bash
make qdrant-down
```

### `make qdrant-logs`

Показывает логи запущенного контейнера Qdrant в реальном времени.

```bash
make qdrant-logs
```

### `make qdrant-build`

Собирает (или загружает) образ контейнера. Обычно выполняется автоматически при первом запуске.

```bash
make qdrant-build
```

Контейнер Qdrant работает в отдельном профиле `qdrant` и управляется командами `make qdrant-up` / `make qdrant-down`.
Он использует отдельное compose-проектное имя (`kilocode`), чтобы запуск для Kilocode был независим от окружения проекта.
