# Ценность (V) — Value

Определяет бизнес-ценность задачи для проекта и пользователей.

## Уровни ценности
- **V0**: Минимальная — опциональные улучшения, можно отложить.
- **V1**: Низкая — косвенная ценность, улучшения "nice-to-have".
- **V2**: Средняя — полезная ценность, улучшает существующие функции.
- **V3**: Высокая — важная ценность для пользователя или ключевой бизнес-процесс.
- **V4**: Критическая — без этого продукта не существует или блокирует бизнес-процессы.
