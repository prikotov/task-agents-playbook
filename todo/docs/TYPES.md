# Типы задач (Префиксы)

Определяет категорию изменений, вносимых задачей. Соответствует стандартам Conventional Commits.

- **fix**: Исправления бага в коде.
- **feat**: Добавление новой функциональности в код.
- **build**: Изменения, влияющие на сборку проекта или зависимости (например, `composer.json`, `Dockerfile`, `Makefile`).
- **chore**: Вспомогательные изменения, не влияющие на исходный код или тесты (например, обновление `.gitignore`).
- **ci**: Изменения в конфигурации CI/CD (например, `.github/workflows`, `.gitlab-ci.yml`).
- **docs**: Изменения только в документации (например, `README.md`, PHPDoc).
- **style**: Изменения, не влияющие на поведение кода: форматирование, пробелы, lint-правки.
- **refactor**: Изменения в коде, улучшающие структуру без изменения поведения.
- **perf**: Изменения, улучшающие производительность (оптимизация запросов, кэширование).
- **test**: Добавление или изменение тестов без изменения исходного кода.
- **revert**: Откат изменений.
- **epic**: Тематический контейнер для связанных задач.
